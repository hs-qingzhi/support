康护APP隐私政策

隐私政策
🔒 最后更新：2026年06月15日

康护APP是一款健康管理应用，用于记录血压、血糖、血脂和BMI数据，提供健康提醒、家庭成员管理、数据统计分析、OCR图片识别等功能。我们十分尊重您的个人信息和健康数据，并会尽全力保护您的信息安全。本隐私政策根据相关法律法规制定，明确说明我们如何收集、使用、存储和保护您的信息。

📊一、收集信息的方式和范围

  • 1.1 健康数据收集：在您使用康护APP记录血压、血糖、血脂和BMI等健康信息时，我们会收集相关数据用于健康分析、趋势提醒和报告生成。您可以随时在APP中查看、修改或删除自己的健康数据。
  • 1.2 基本设备信息：为了提供更稳定和个性化的服务，我们可能会在您同意隐私政策后收集设备硬件信息（如设备型号、操作系统版本等）及网络信息（如 IP 地址、网络类型），用于统计、安全风控和优化服务体验。
  • 1.3 第三方服务使用：部分功能依赖第三方服务来实现。只有在您同意隐私政策后，相关服务才会运行。具体包括：
    • 友盟统计服务（Umeng）：用于 APP 使用情况统计、性能优化与崩溃监控。服务收集设备信息（设备型号、操作系统版本、网络类型、IP 地址、传感器列表、AndroidID）和事件数据（如功能使用、订阅转化、付费墙触达），但不会收集您的健康数据明细。
    • 友盟推送服务（Umeng Push）：用于向您发送健康提醒和重要通知，使用 APNs/Push token 等设备识别信息。
    • 百度 OCR 服务（Baidu Cloud OCR）：用于识别血压计、血糖仪等健康设备屏幕上的数据。当您使用拍照识别功能时，您拍摄的照片会上传至百度云服务器进行 OCR 处理，识别完成后图片不会保留在我们的服务器，但请避免在照片中包含与健康设备无关的敏感信息（如身份证、面部、银行卡等）。
    • Firebase（Google 提供）：包括 Firebase Cloud Messaging（推送通知）和 Firebase Remote Config（远程配置）。可能收集设备识别信息、IP 地址和 App 安装信息。
    • Google AdMob 广告 SDK（仅国际版/海外渠道）：在免费用户开屏页展示广告。AdMob 可能收集广告标识符（IDFA / IDFV）、IP 地址、设备信息和广告交互数据。订阅 Premium 用户不会展示广告。
    • 字节跳动穿山甲（Pangle / GroMore 聚合）SDK（仅国内渠道）：在免费用户开屏页展示广告。可能收集网络信息（IP 地址）、传感器信息（传感器列表，用于广告反作弊与设备识别）及设备标识信息（OAID 等；MAC/IMEI 已禁用，仅基础识别）。订阅 Premium 用户不会展示广告。
    • HTTPDNS-android（开发者：杭州阿里云智能科技有限公司，仅国内渠道）：由统计/推送等第三方 SDK 依赖集成，用于域名解析优化、防止 DNS 劫持，保障网络请求的稳定与安全。可能收集网络信息（IP 地址）。SDK 隐私政策：https://terms.aliyun.com/legal-agreement/terms/suit_bu1_ali_cloud/suit_bu1_ali_cloud202112071754_83380.html
    • App 内购买（StoreKit）：通过 Apple 提供的 In-App Purchase 框架处理订阅与支付，由 Apple 直接管理交易；我们仅接收订阅状态和到期时间，不接触您的支付凭据。
  • 1.4 用户反馈与投诉：您可以在APP“我的”页面中的“问题反馈”功能提交反馈或投诉，涉及个人信息或健康数据时，我们会严格按照隐私政策处理。

💡二、信息使用

  • 提供健康数据记录、分析与趋势提醒服务。
  • 优化产品功能与用户体验，包括性能监控与稳定性改进。
  • 向您发送重要健康提醒、服务通知与必要的运营消息。
  • 开展产品使用情况统计与分析，以改进服务质量与可靠性。
  • 处理用户反馈、问题定位与技术支持请求。
  • PDF 健康报告：您可主动将健康记录在设备本地生成 PDF 文件，并通过系统分享功能自主分享（如微信、邮件）。文件仅在本地生成，应用不会将该文件上传至任何服务器。

🛡️三、信息存储与保护

  • 健康数据会加密存储，并采取合理的技术措施防止未经授权的访问、泄露或篡改。
  • 严格限制员工对个人信息的访问权限，仅授权必要岗位处理相关数据。
  • 定期开展安全审计与漏洞检测，及时修复安全问题并加强防护。
  • 我们承诺不向无关第三方出售、共享或泄露您的健康数据，除非法律法规要求或为保障用户安全。
  • 安全声明：我们采取合理、必要的技术与管理措施，但无法保证信息绝对安全。用户理解并同意，因不可抗力、第三方原因或用户自身原因导致的数据丢失、泄露或损坏，公司在法律允许范围内不承担赔偿责任。

🔐四、设备权限使用说明

我们会在必要时向您申请以下设备权限，您可以选择同意或拒绝授权：

  • 4.1 相机权限
    • 用途：①为家庭成员拍摄头像照片，方便识别和管理不同成员的健康数据；②在记录血压、血糖、血脂时，拍照识别（OCR）血压计、血糖仪等设备屏幕上的读数，便于快速录入。
    • 使用场景：仅在您点击“添加成员 → 拍照”，或在记录表单中使用“拍照识别”功能时申请。
    • 您的选择：您可以拒绝授权并改用手动输入，或从相册选择照片。
  • 4.2 通知权限
    • 用途：发送健康数据记录提醒、用药提醒等健康管理通知，帮助您保持健康习惯。
    • 使用场景：仅在您主动设置提醒时使用。
    • 您的选择：您可以随时在系统设置中关闭通知权限。
  • 4.3 隐私保护承诺
    • ✅ 康护APP不会申请存储权限，照片均通过系统相册选择器完成，保护您的文件隐私。
    • ✅ 健康数据默认仅保存在您的设备本地；仅当您主动开启“云备份”功能时，相关健康数据才会通过加密通道（HTTPS）上传至我们的服务器，用于备份与多设备同步，您可随时关闭云备份并删除云端数据。
    • ✅ 我们不会收集您的位置信息、通讯录、通话记录等敏感信息。

⚖️五、用户权利

  • 您有权访问、修改或删除自己的健康数据。
  • 您可以选择退出部分数据收集或功能，但可能影响某些服务体验。
  • 在法律允许范围内，您可随时撤回对隐私政策的同意。

🔄六、政策更新

  • 我们可能会根据法律法规及产品更新对本隐私政策进行修改，并在APP中公告更新版本。我们将尽合理努力通知，但无需逐一取得同意。
  • 继续使用APP即表示您同意更新后的隐私政策。

🧩七、第三方服务声明

  • 部分功能依赖第三方服务（如统计、推送、OCR等）。第三方服务的数据处理规则以其隐私政策为准；因第三方服务导致的争议由该第三方承担责任。
  • 公司在法律允许范围内不对第三方服务的可用性、准确性、合规性作出保证或承担连带责任。
  • OCR/识别功能可能存在误识别或延迟，仅供参考，请以人工确认为准。

🗄️八、数据保留与备份

  • 我们将基于实现目的所必需的最短期限保存您的个人信息，超期将进行删除或匿名化处理，法律另有规定的除外。
  • 建议用户定期自行备份数据；因用户未备份或设备问题导致的数据丢失，公司在法律允许范围内不承担赔偿责任。

💬九、意见反馈功能的信息处理

当您使用 App 内「意见与反馈」时，我们会收集并处理以下信息，仅用于接收与处理您的反馈、与您沟通、改进产品：

  • 9.1 您主动提供的信息：反馈内容、图片、联系方式（选填）。其中联系方式经加密后存储。
  • 9.2 为定位问题自动附带的信息：App 版本、系统版本、设备型号、语言、会员状态、所在页面，以及一个匿名设备标识（不含您的姓名、手机号等真实身份）；提交时还会记录访问 IP，仅用于防滥用与安全防护，不作长期留存。
  • 9.3 存储与第三方：数据存储于阿里云（中国境内，北京）。图片上传至阿里云对象存储（OSS），并经阿里云 DashScope 视觉模型进行机器审核以拦截违规内容，该图片审核为我们对阿里云的委托处理。
  • 9.4 留存期限：图片默认保留 12 个月后自动删除；文字反馈在工单解决后保留 12 个月，逾期自动清理。
  • 9.5 您的权利：您可在「我的反馈」内删除您的全部反馈（连同图片），提交后即时移除记录、图片随后清理。
  • 9.6 非公开：本渠道为您与开发者之间的非公开支持工单，内容不会公开展示给其他用户。
